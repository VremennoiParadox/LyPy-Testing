# LyPy — Lyrics overlay

A lightweight, always-on-top desktop widget that shows **time-synced lyrics** for whatever you are playing on Windows, in real time.

![Python](https://img.shields.io/badge/Python-3.10+-blue)
![PyQt5](https://img.shields.io/badge/GUI-PyQt5-green)
![Windows](https://img.shields.io/badge/OS-Windows%2010%2F11-blue)

---

## Supported music apps

LyPy uses the **Windows Media Transport Controls** (WMTC) API. Any player that registers a session works, including:

- **Spotify**, **YouTube Music**, **Apple Music**, **Amazon Music**, **Tidal**, **Deezer**, **Yandex Music**
- Desktop browsers that expose WMTC (for example Microsoft Edge, Google Chrome, Mozilla Firefox)

> If two or more apps are playing at the same time, LyPy shows a warning so you can pause extras and avoid bad sync.

---

## Features

- **Live lyrics** with line highlighting (timed LRC from LRCLIB when available)
- **Single lyrics source** — [LRCLIB](https://lrclib.net/) (large community database, synced + plain)
- **No streaming login** — playback comes from Windows; lyrics are fetched over HTTPS
- **Tray icon**, always-on-top overlay, pin, resize, album-art gradient
- **Optional behaviour** — start at Windows login, start hidden in tray, show when Spotify is the active WMTC session, raise when `Spotify.exe` starts (LyPy must already be running; see [Spotify startup](#spotify-startup-what-works-and-what-does-not))

---

## How it works

1. **WMTC** reports title, artist, album, position, and play state.
2. **Lyrics** — [LRCLIB](https://lrclib.net/) only: exact match by track/artist/album, then search. Synced LRC is preferred over plain text so highlighting matches playback when timestamps exist.
3. The overlay scrolls to the active line (LRC timestamps, or an approximate position for plain lyrics).

## Setup (from source)

### 1. Install dependencies

```bash
pip install -r LyPy/requirements.txt
```

### 2. (First time) Generate UI assets

Icons live under `LyPy/assets/`. If the folder is missing:

```bash
cd LyPy
py -3 generate_icons.py
```

### 3. Run

```bash
python LyPy/main.py
```

## Release (Windows `.exe`)

End users run **`dist/LyPy.exe`** only — no Python install required. Builds are **Windows-only** (WinRT + PyInstaller onefile).

### Prerequisites (maintainers)

- Windows 10 or 11
- Python 3.10+ on PATH (`py -3` or `python`)
- **Bundled font:** place `Nunito-Variable.ttf` at `LyPy/assets/fonts/Nunito-Variable.ttf` ([Nunito on Google Fonts](https://fonts.google.com/specimen/Nunito), OFL). Icons are generated by script; PNGs are not always committed.

### Build commands

**Recommended** — full pipeline (deps, icons, font check, PyInstaller):

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\build_windows.ps1
```

**Manual** steps from the repository root:

```powershell
pip install -r LyPy/requirements.txt
pip install -r requirements-dev.txt
cd LyPy
py -3 generate_icons.py
cd ..
py -3 build_exe.py
```

Output: **`dist/LyPy.exe`** (~80–150 MB onefile). Bundled inside: `assets/` (icons + fonts), `scripts/` (e.g. logon task installer), PyQt5 plugins (`--collect-all PyQt5`), and WinRT (`--collect-all winrt`).

`LyPy.spec` is **gitignored** — PyInstaller may write machine-specific absolute paths there. Do not commit it; regenerate with `build_exe.py` on each machine.

Close any running **LyPy.exe** (including tray) before rebuilding, or Windows will block overwriting `dist/LyPy.exe`.

### First run (end users)

- Settings are stored at **`%LOCALAPPDATA%\LyPy\settings.json`** (created on first save). Window size, API keys, and behaviour flags persist across updates if you keep that file.
- **Windows SmartScreen** may warn on an unsigned `.exe` from GitHub or a zip. Choose “More info” → “Run anyway”, or sign the binary with a code-signing certificate for distribution.

### Release smoke test (clean VM)

- Launch `LyPy.exe` without Python installed; play music in Spotify; lyrics and tray work.
- Toggle **Start at Windows login** (registry) or run `scripts/install_spotify_hook.ps1` with your exe path.
- Confirm lyrics load from LRCLIB (synced tracks should highlight in time with playback).

## Spotify startup (what works and what does not)

LyPy does **not** hook into Spotify’s installer and does **not** cold-launch itself when you open Spotify unless you add separate Windows automation (Task Scheduler on process start, WMI, etc.). That pattern is **not built in** and is not documented here as a supported product feature.

### Truth table

| Approach | Launches LyPy when Spotify opens? | Built into LyPy? |
|----------|-------------------------------------|------------------|
| `raise_on_spotify_process_start` | **No** — only raises LyPy if it is **already running** (for example in the tray) | Yes |
| `auto_show_on_spotify` | **No** — shows/raises when Spotify is the **active WMTC session** while LyPy is running | Yes |
| `start_at_login` or [install_spotify_hook.ps1](scripts/install_spotify_hook.ps1) | **No** — starts LyPy at **Windows user logon**, not when Spotify starts | Yes |
| Cold-launch when `Spotify.exe` starts | Possible with external Task Scheduler / WMI | **Not implemented** |

### Recommended pattern (tray + appear when you open Spotify)

For behaviour closest to “LyPy follows Spotify” without external hooks:

1. **Start LyPy at Windows login** (`start_at_login`)
2. **Launch hidden to tray** (`start_hidden`)
3. **Raise LyPy when Spotify.exe starts** (`raise_on_spotify_process_start`) — and/or **Show overlay when Spotify is the active player** (`auto_show_on_spotify`)

LyPy stays in the tray after logon and shows or raises when you launch Spotify or when Spotify becomes the active media session.

### `auto_show_on_spotify` vs `raise_on_spotify_process_start`

These are easy to mix up; both require **LyPy to already be running** (they do not start LyPy from a fully quit state).

| Setting | Trigger | What happens |
|---------|---------|----------------|
| **`auto_show_on_spotify`** | Spotify becomes the **active Windows Media Transport Controls (WMTC) session** — you are playing in Spotify and Windows reports it as the current player | Show or raise the overlay |
| **`raise_on_spotify_process_start`** | Windows detects **`Spotify.exe` starting** (process appeared; poll in `spotify_process.py`) | Raise LyPy if it is already running (for example in the tray) |

Use **auto-show** when you care about “Spotify is the player I’m listening to.” Use **raise on process start** when you care about “I just launched the Spotify app.” Many users enable both together with login + start hidden.

### Start at logon (script)

[scripts/install_spotify_hook.ps1](scripts/install_spotify_hook.ps1) registers a **user logon** scheduled task. The filename is historical — it is **not** a Spotify hook and does not run when Spotify opens. Edit the path to `LyPy.exe`, then run in PowerShell:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\install_spotify_hook.ps1 -LyPyExe "C:\path\to\LyPy.exe"
```

The in-app **Start LyPy at Windows login** setting writes the same user Run key; you only need the script if you prefer a scheduled task over the registry.

## Usage

1. Play music in a supported app.
2. LyPy shows **Fetching lyrics…** then the best result it found.
3. Drag the title bar, resize from edges, use pin to lock position.
4. Settings (gear): text, behaviour, optional API keys, bug-report URL.

Settings file: `%LOCALAPPDATA%\LyPy\settings.json`.

| Setting | Default | Description |
|---------|---------|-------------|
| `window_width` / `window_height` | 500 / 700 | Window size |
| `window_opacity` | 1.0 | 0.3 – 1.0 |
| `font_size` | 28 | Lyrics size |
| `font_family` | Segoe UI stack | Font CSS |
| `bg_saturation` | 80 | Gradient saturation |
| `line_spacing` | 3 | Line gap |
| `polling_interval_ms` | 50 | Playback poll interval |
| `scroll_animation_ms` | 400 | Scroll animation length |
| `always_on_top` | true | Stay on top |
| `start_at_login` | false | Run at user logon (registry) |
| `auto_show_on_spotify` | true | While LyPy is running: show/raise when Spotify is the active WMTC session (does not cold-launch LyPy) |
| `raise_on_spotify_process_start` | false | While LyPy is running: raise when `Spotify.exe` starts (does not cold-launch LyPy) |
| `start_hidden` | false | Start minimized to tray |
| `bug_report_url` | `https://github.com` | URL opened by “Report a bug” |

## Troubleshooting

| Problem | What to try |
|---------|-------------|
| “Play something…” but music is playing | Prefer the desktop app; some browser tabs do not publish WMTC. |
| “No lyrics available” | LRCLIB may not have that track; try again later or check spelling on lrclib.net. |
| Multi-app warning | Pause other players so only one is playing. |
| `winrt` install fails | Windows 10+ and a supported Python (3.10+). |
| Frozen exe issues | Rebuild: `.\scripts\build_windows.ps1` or `py -3 build_exe.py`; ensure `LyPy/assets/fonts/Nunito-Variable.ttf` exists. |
